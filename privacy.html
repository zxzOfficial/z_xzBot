01 — Introduction

Introduction

Cette Politique de Confidentialité décrit précisément comment z_xzbot collecte, utilise et protège les données lors de son utilisation sur Discord. Elle est basée sur le code source réel du bot.

Nous nous engageons à ne collecter que les données strictement nécessaires au fonctionnement du bot. Aucune donnée n'est vendue, partagée ou transmise à des tiers.

02 — Fichiers de données

Fichiers JSON utilisés par le bot

z_xzbot utilise 8 fichiers JSON stockés localement dans le dossier data/ :

Fichier Contenu Utilisé par
warns.json Avertissements par utilisateur et serveur /warn, /warns, /clearwarns
config.json Configuration de chaque serveur (rôles, salons, paramètres) /config, tous les modules
tickets.json Tickets ouverts : ID salon, ID utilisateur, auteur Système de tickets
mutes.json Mutes temporaires : clé guild:user → timestamp d'expiration /mute, tâche automod_cleanup
history.json Historique des actions de modération par membre /historique, toutes les actions mod
notes.json Notes privées des modérateurs sur les membres /note, /notes
servers.json Nom, membres, owner, date de join/leave pour chaque serveur on_guild_join/remove, panel.py
giveaways.json Lot, salon, fin, nb gagnants, organisateur, statut /giveaway, tâche giveaway_task

04 — Utilisation

Utilisation des données

Les données sont utilisées exclusivement pour :

  • Appliquer la configuration de chaque serveur (rôles, salons, seuils)
  • Gérer les avertissements et déclencher les sanctions automatiques (timeout, ban)
  • Faire fonctionner le système de tickets et envoyer les transcripts
  • Tracer les actions de modération dans les logs et l'historique
  • Gérer les giveaways et les tirages au sort
  • Alimenter le panneau de contrôle panel.py avec les statistiques des serveurs

Aucune donnée n'est vendue, partagée, ni transmise à un tiers.

05 — Stockage & sécurité

Stockage et sécurité

Toutes les données sont stockées localement sur la machine hébergeant le bot, dans des fichiers JSON du dossier data/. Aucune base de données cloud, aucun service tiers n'est utilisé. L'accès est limité à l'opérateur de l'instance du bot.

Les logs du bot sont stockés dans logs/bot.log avec rotation automatique (5 Mo max, 5 fichiers de sauvegarde).

06 — Conservation

Conservation des données

Les données d'un serveur sont conservées tant que le bot y est présent. Lorsque le bot quitte un serveur, les données sont marquées comme inactives mais restent présentes sauf demande de suppression.

Les données d'un utilisateur (warns, notes, historique) peuvent être effacées via :

  • La commande /clearwarns (warns uniquement)
  • Demande directe au développeur sur le serveur de support
07 — Vos droits

Vos droits (RGPD)

Conformément au Règlement Général sur la Protection des Données (RGPD), vous disposez des droits suivants :

  • Droit d'accès — connaître les données stockées vous concernant
  • Droit de rectification — corriger des données inexactes
  • Droit à l'effacement — supprimer vos données (warns, notes, historique)
  • Droit d'opposition — vous opposer au traitement de vos données

Pour exercer ces droits, contactez le développeur directement via le serveur de support Discord. Toute demande sera traitée dans un délai raisonnable.
